# Plan: Update Execute Skill — Skip CI for Docs-Only Plans & Update Review Files Post-Execution

## Context

Two behavioral gaps in the PMP execute skill:
1. CI runs even when a plan only modifies markdown/architectural files — wasteful
2. After executing fixes from a review discussion, the review files are never updated to reflect what was fixed/addressed/acknowledged and the decisions made

## Changes

### 1. Skip CI for documentation-only plans

**File:** [execute-loop.md](plugins/pmp/skills/execute/references/execute-loop.md)

Add a detection step early in execution (after reading the plan) that checks whether the plan only touches markdown/docs/architectural files. If so, skip all CI gates (Task A CI, Task B CI, Final CI). This applies to:
- Plans generated by `/pmp:discuss` that only fix specs/docs
- Any plan where all features are spec/doc changes

Add a condition before each CI gate: "If docs-only plan, skip CI."

### 2. Update review files after plan execution

**File:** [execute-loop.md](plugins/pmp/skills/execute/references/execute-loop.md)

Add a new step in the Completion Phase (after archiving the plan, before announcing completion):
- Check if the plan has a `source_review` field in frontmatter (set by discuss-generated plans)
- If yes, read the source review file
- Append a `## Resolution` section to the review file documenting:
  - Which findings were fixed (with brief description of the fix)
  - Which findings were acknowledged (with rationale)
  - Which findings were deferred (with reason)
  - Date of resolution

**File:** [discuss.md](plugins/pmp/skills/discuss/references/discuss.md)
- Remove the "DO NOT modify the review file" constraint — it should be modifiable post-execution
- Or clarify: "Do not modify the review file during discussion; it will be updated after execution"

## Verification

- Read the modified files and confirm the logic is coherent
- No code changes, so no CI needed
